pay_rewards ctxt ?active_stake source delegate transfers the rewards to the delegate spendable balance and frozen balance.

The distribution is based on the baker's staking parameters.

If adaptive issuance is enabled, it also accounts for the repartition of the delegate's active_stake between delegated token and frozen deposits. If active_stake is not provided, it will be retrieved from the context.
