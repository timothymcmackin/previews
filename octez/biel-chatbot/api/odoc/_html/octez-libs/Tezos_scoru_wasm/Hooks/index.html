Module Tezos_scoru_wasm.Hooks

type t = {
  1. fast_exec_invalid_kernel : [ `Check_with_hook of (unit -> unit Lwt.t) option | `No_check ];
  2. fast_exec_panicked : (exn -> unit Lwt.t) option;
  3. fast_exec_completed : (unit -> unit Lwt.t) option;
  4. fast_exec_fallback : bool;
}

A collection of hooks to personalize the execution of the WASM Fast Execution.

val no_hooks : t

no_hooks is the empty collection of hooks, which can be used as a baseline to build hooks collection.

val disable_fast_exec_invalid_kernel_check : t -> t

disable_fast_exec_invalid_kernel_check hooks returns a new set of hooks signaling to the fast execution runtime to disable the sanity check of the kernel it is trying to load.

val on_fast_exec_invalid_kernel : (unit -> unit Lwt.t) -> t -> t

on_fast_exec_invalid_kernel k hooks replaces b the hook executed when the Fast Execution engine is requested to run a kernel not satisfying the WASM PVM constraints.

val on_fast_exec_panicked : (exn -> unit Lwt.t) -> t -> t

on_fast_exec_panicked k hooks replaces b the hook executed when the Fast Execution engine panics by k in hooks.

val on_fast_exec_completed : (unit -> unit Lwt.t) -> t -> t

on_fast_exec_completed k hooks replaces b the hook executed when the Fast Execution engine completes a kernel_run by k in hooks.

val fast_exec_fallback : bool -> t -> t

fast_exec_fallback c hooks returns a new collection of hooks instructing the Fast Execution engine to fallback to the WASM PVM in case of errors.

It is true by default, but keep in mind the Fast Execution is very b slow.
